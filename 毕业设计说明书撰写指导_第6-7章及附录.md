### 6.3 性能测试

#### 6.3.1 响应时间测试

**测试方法**：
使用Postman对核心API接口进行响应时间测试，每个接口测试100次取平均值。

**测试结果**：

| 接口名称 | 请求方法 | 平均响应时间(ms) | 最大响应时间(ms) | 最小响应时间(ms) | 成功率 |
|---------|---------|----------------|----------------|----------------|--------|
| 获取菜地列表 | GET /api/gardens | 125 | 280 | 85 | 100% |
| 获取菜地详情 | GET /api/gardens/{id} | 95 | 150 | 60 | 100% |
| 创建订单 | POST /api/orders | 180 | 320 | 120 | 100% |
| 获取订单列表 | GET /api/orders | 140 | 250 | 95 | 100% |
| 获取帖子列表 | GET /api/posts | 160 | 300 | 110 | 100% |
| 创建帖子 | POST /api/posts | 200 | 350 | 140 | 100% |
| 点赞 | POST /api/posts/{id}/like | 80 | 120 | 50 | 100% |
| 添加评论 | POST /api/posts/{id}/comments | 110 | 180 | 75 | 100% |

**性能评估**：
- ✅ 所有接口平均响应时间均小于500ms，满足性能要求
- ✅ 读操作（GET）响应时间优于写操作（POST）
- ✅ 成功率达到100%，系统稳定性良好

#### A.1 小程序API封装

**文件：miniprogram/api/request.js**
```javascript
const baseURL = 'http://localhost:8000/api'

function request(options) {
  return new Promise((resolve, reject) => {
    const token = wx.getStorageSync('token')
    
    wx.request({
      url: baseURL + options.url,
      method: options.method || 'GET',
      data: options.data || {},
      header: {
        'Content-Type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : ''
      },
      success: (res) => {
        if (res.statusCode === 200) {
          resolve(res.data)
        } else if (res.statusCode === 401) {
          // token过期，跳转登录
          wx.redirectTo({ url: '/pages/login/login' })
          reject(new Error('未登录'))
        } else {
          reject(new Error(res.data.detail || '请求失败'))
        }
      },
      fail: reject
    })
  })
}

module.exports = { request }
```

from sqlalchemy.sql import func
from app.core.database import Base

class Garden(Base):

    area = Column(Numeric(10, 2), nullable=False)
    price = Column(Numeric(10, 2), nullable=False)
