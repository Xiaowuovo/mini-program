## 🎯 项目核心信息总结

### 项目背景
在城市化进程加速的背景下，都市居民对绿色生活和自然体验的需求日益增长。共享菜园作为一种新兴的共享经济模式，让城市居民能够租用菜地体验种植乐趣，但面临着无法实时照看菜地的痛点。本项目旨在开发一款专属小程序，通过远程监控、智能提醒等功能，解决租户与菜园管理方的信息不对称问题。

### 技术架构
- **前端**：微信小程序原生框架（WXML + WXSS + JavaScript）
- **后端**：Python 3.12 + FastAPI + SQLAlchemy
- **数据库**：MySQL 8.0（14张核心表）
- **硬件集成**：树莓派 + USB摄像头（远程监控方案）
- **部署架构**：前后端分离，RESTful API设计

### 核心功能模块（7大模块）
1. **用户认证系统**：微信一键登录、用户信息管理
2. **菜地管理模块**：菜地浏览、筛选、详情展示、在线租用
3. **订单管理系统**：订单创建、支付、状态追踪
4. **远程监控功能**：基于树莓派的视频流传输
5. **智能提醒系统**：基于作物生长规则的自动提醒生成
6. **社区互动平台**：种植动态分享、点赞评论
7. **增值服务模块**：代浇水、代施肥等服务预约

### 项目规模统计
- **前端页面**：27个页面（含管理端5个）
- **后端API**：50+ 个接口
- **数据库表**：14张表
- **代码总量**：约18,000行
- **开发周期**：3-4个月
- **当前完成度**：70%

## 🔧 第二章：相关技术与理论基础

### 2.1 微信小程序技术

#### 2.1.1 小程序概述
微信小程序是一种无需下载安装即可使用的轻量级应用，基于微信生态提供丰富的原生能力和开放接口。

#### 2.1.2 小程序框架
- **WXML**：类似HTML的标记语言
- **WXSS**：类似CSS的样式语言
- **JavaScript**：小程序逻辑层语言
- **组件系统**：丰富的内置组件

#### 2.1.3 小程序能力
- 微信登录与用户信息获取
- 本地存储
- 网络请求
- 媒体能力（图片、视频）
- 位置服务
- 订阅消息

### 2.2 后端技术

#### 2.2.1 FastAPI框架
- 基于Python 3.6+的现代Web框架
- 高性能（基于Starlette和Pydantic）
- 自动生成API文档（Swagger UI）
- 类型提示和数据验证
- 异步支持

#### 2.2.2 SQLAlchemy ORM
- Python最流行的ORM工具
- 支持多种数据库
- 提供声明式和经典映射方式
- 查询优化和连接池管理

#### 2.2.3 数据库技术
- **MySQL 8.0**：开源关系型数据库
- **JSON字段**：支持半结构化数据存储
- **索引优化**：提升查询性能

### 2.3 物联网技术

#### 2.3.1 树莓派
- 低成本单板计算机
- 支持Linux操作系统
- GPIO接口丰富

#### 2.3.2 视频采集与传输
- USB摄像头驱动（V4L2）

#### 2.3.3 传感器技术
- 土壤湿度传感器
- 温湿度传感器

## 📊 第三章：系统需求分析

### 3.1 业务需求分析

#### 3.1.1 租户需求分析

**核心需求**：
1. **菜地选择**：浏览可租用菜地，查看详细信息（面积、价格、位置、图片）
2. **在线租用**：选择租期、在线下单、支付租金
3. **远程监控**：实时查看菜地生长情况
4. **智能提醒**：接收浇水、施肥、收获等提醒
5. **社区交流**：分享种植经验、查看他人动态
6. **增值服务**：预约代浇水、代施肥等服务

**次要需求**：
- 个人信息管理
- 订单历史查询
- 帮助文档查看

#### 3.1.2 管理方需求分析

**核心需求**：
1. **菜地管理**：添加、编辑、删除菜地信息
2. **订单管理**：查看所有订单、更新订单状态
3. **服务工单**：处理用户服务请求
4. **用户管理**：查看用户信息、处理问题

**次要需求**：
- 数据统计分析
- 收入报表生成

#### 3.1.3 双边平台需求特点

**平台核心价值**：
- 为租户提供便捷的菜地租用和管理渠道
- 为管理方提供数字化管理工具
- 降低双方沟通成本，提高效率

**平台运营需求**：
- 确保信息真实可靠
- 促进用户活跃和留存
- 优化服务质量

#### 3.2.2 核心用例描述

**用例1：菜地租用**
- **用例名称**：菜地租用
- **参与者**：租户
- **前置条件**：用户已登录
- **主要流程**：
  1. 用户浏览菜地列表
  2. 筛选符合条件的菜地
  3. 点击查看菜地详情
  4. 选择租用日期和租期
  5. 确认订单信息
  6. 完成支付
  7. 系统生成订单
- **后置条件**：订单创建成功，菜地状态更新为已租用

**用例2：智能提醒**
- **用例名称**：智能提醒
- **参与者**：系统、租户
- **前置条件**：用户已租用菜地并添加种植记录
- **主要流程**：
  1. 系统根据作物生长规则计算当前阶段
  2. 生成相应的提醒任务（浇水、施肥等）
  3. 在提醒时间推送消息给用户
  4. 用户查看提醒列表
  5. 用户标记提醒为已完成
- **后置条件**：提醒状态更新

### 3.3 非功能性需求

#### 3.3.1 性能需求
- **响应时间**：API接口平均响应时间 < 500ms
- **并发处理**：支持至少100个并发用户
- **页面加载**：小程序页面加载时间 < 2秒

#### 3.3.2 安全性需求
- **身份认证**：基于微信OpenID的安全登录
- **数据传输**：HTTPS加密传输
- **权限控制**：区分普通用户和管理员权限
- **数据备份**：定期备份数据库

#### 3.3.3 可用性需求
- **系统可用性**：7×24小时运行，可用率 > 99%
- **错误恢复**：系统故障后能够自动恢复
- **容错设计**：异常情况下给予用户友好提示

#### 3.3.4 可维护性需求
- **代码规范**：遵循PEP 8（Python）和微信小程序开发规范
- **文档完整**：提供API文档、部署文档、用户手册
- **模块化设计**：功能模块解耦，便于维护和扩展
