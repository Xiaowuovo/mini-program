#### 4.1.2 技术选型论证

**前端技术选型**：

| 方案 | 优势 | 劣势 | 评分 |
|------|------|------|------|
| 微信小程序原生 | 性能好、能力丰富、用户基数大 | 只能在微信内使用 | ⭐⭐⭐⭐⭐ |
| uni-app | 跨平台 | 性能略差、调试复杂 | ⭐⭐⭐ |
| React Native | 技术成熟 | 体积大、不适合轻量应用 | ⭐⭐ |

**选择结论**：微信小程序原生框架（最佳匹配项目需求）

**后端技术选型**：

| 方案 | 优势 | 劣势 | 评分 |
|------|------|------|------|
| FastAPI | 高性能、自动文档、现代化 | 相对较新 | ⭐⭐⭐⭐⭐ |
| Django | 功能全面、生态丰富 | 重量级、性能一般 | ⭐⭐⭐⭐ |
| Flask | 轻量灵活 | 需要自行选择组件 | ⭐⭐⭐ |
| Node.js | 前后端统一语言 | 异步编程复杂 | ⭐⭐⭐ |

**选择结论**：FastAPI（性能优异、开发效率高、自动生成API文档）

**数据库选型**：

| 方案 | 优势 | 劣势 | 评分 |
|------|------|------|------|
| MySQL 8.0 | 成熟稳定、JSON支持、社区活跃 | - | ⭐⭐⭐⭐⭐ |
| PostgreSQL | 功能强大 | 配置复杂 | ⭐⭐⭐⭐ |
| MongoDB | 灵活的文档模型 | 不适合关系型数据 | ⭐⭐⭐ |

**选择结论**：MySQL 8.0（适合关系型数据、支持JSON字段、运维成本低）

#### 5.1.1 用户认证模块

**功能描述**：
实现基于微信OpenID的一键登录，无需用户输入账号密码。

**技术流程**：
```
1. 小程序调用wx.login()获取临时code
2. 将code发送到后端
3. 后端用code换取session_key和openid
4. 查询或创建用户记录
5. 生成JWT token返回给前端
6. 前端存储token用于后续API请求
```

**关键代码**：

前端登录代码（`miniprogram/pages/login/login.js`）：
```javascript
// 微信登录
wechatLogin() {
  wx.login({
    success: (res) => {
      if (res.code) {
        // 调用后端登录接口
        authAPI.wechatLogin(res.code).then(result => {
          // 保存token
          wx.setStorageSync('token', result.access_token)
          wx.setStorageSync('userInfo', result.user)
          // 跳转首页
          wx.switchTab({ url: '/pages/index/index' })
        })
      }
    }
  })
}
```

后端登录接口（`backend/app/api/auth.py`）：
```python
@router.post("/wechat-login")
async def wechat_login(code: str, db: Session = Depends(get_db)):
    # 1. 用code换取openid
    openid = await get_openid_from_wechat(code)
    
    # 2. 查询或创建用户
    user = db.query(User).filter(User.openid == openid).first()
    if not user:
        user = User(openid=openid)
        db.add(user)
        db.commit()
    
    # 3. 生成JWT token
    token = create_access_token(data={"user_id": user.id})
    
    return {
        "access_token": token,
        "user": UserSchema.from_orm(user)
    }
```

```python
@router.post("/orders")
async def create_order(
    order_in: OrderCreate,
    current_user: User = Depends(get_current_user),
    db: Session = Depends(get_db)
):
    # 1. 检查菜地是否可用
    garden = db.query(Garden).filter(Garden.id == order_in.garden_id).first()
    if garden.status != "available":
        raise HTTPException(400, "菜地不可用")
    
    # 2. 计算租期和总价
    days = (order_in.end_date - order_in.start_date).days
    months = days / 30
    total_price = garden.price * months
    
    # 3. 创建订单
    order = Order(
        user_id=current_user.id,
        garden_id=order_in.garden_id,
        start_date=order_in.start_date,
        end_date=order_in.end_date,
        total_price=total_price,
        status="pending"
    )
    db.add(order)
    
    # 4. 更新菜地状态为已租用
    garden.status = "rented"
    
    db.commit()
    
    return OrderSchema.from_orm(order)
```

### 5.2 远程监控模块设计与实现

#### 5.2.1 视频方案选型分析

**方案对比**：

| 方案 | 优势 | 劣势 | 成本 | 适用场景 |
|------|------|------|------|----------|
| 树莓派+FFmpeg推流 | 成本低、可控性强 | 需要自建流媒体服务器 | ¥300 | 小规模部署 |
| 腾讯云TRTC | 实时性好、稳定 | 成本高 | ¥0.05/分钟 | 商业化项目 |
| web-view+云视频 | 开发简单 | 灵活性差 | 中等 | 快速实现 |
| 第三方监控摄像头 | 即插即用 | 不可定制 | 高 | 快速上线 |

**本项目选择**：树莓派+定时快照方案
- 成本低（单个监控点约300元）
- 满足基本监控需求
- 可扩展为实时推流

#### 5.2.2 树莓派硬件集成方案

**硬件配置**：
- 树莓派Zero 2W或4B（150-300元）
- USB摄像头（80元）
- TF卡32GB（40元）
- 防水外壳（30元）

**软件部署**：
1. 安装Raspberry Pi OS Lite
2. 配置WiFi和SSH
3. 安装FFmpeg、Python
4. 部署监控脚本


### 5.3 智能提醒模块设计与实现

#### 5.3.1 提醒机制设计

**提醒类型**：
1. **手动提醒**：用户自定义创建
2. **智能提醒**：基于作物生长规则自动生成
3. **物联网提醒**：基于传感器数据触发（预留）

**提醒生成流程**：
```
1. 用户添加种植记录（作物类型、种植日期）
2. 系统查询该作物的生长阶段规则
3. 根据当前日期计算生长阶段
4. 根据阶段规则生成提醒任务
5. 在提醒时间推送消息给用户
```

**智能提醒生成算法**：

```python
def generate_smart_reminders(planting_record_id):
    """为种植记录生成智能提醒"""
    
    # 1. 获取种植记录
    record = db.query(PlantingRecord).get(planting_record_id)
    
    # 2. 计算当前生长阶段
    days_since_planting = (datetime.now() - record.planting_date).days
    current_stage = calculate_growth_stage(record.crop_id, days_since_planting)
    
    # 3. 获取该阶段的规则
    stage_rule = db.query(CropGrowthStage).filter(
        CropGrowthStage.crop_id == record.crop_id,
        CropGrowthStage.stage == current_stage
    ).first()
    
    # 4. 生成浇水提醒
    if stage_rule.watering_frequency:
        create_watering_reminder(record, stage_rule)
    
    # 5. 生成施肥提醒
    if stage_rule.fertilizing_frequency:
        create_fertilizing_reminder(record, stage_rule)
    
    # 6. 生成其他提醒
    if stage_rule.weeding_frequency:
        create_weeding_reminder(record, stage_rule)
```
